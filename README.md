# GarXmlParser

# Область применения:

	- Репликатор предназначен для работы с полными выгрузками адресного реестра ФНС в формате "ГАР".
	- Реализована фильтрация по регионам на уровне имен папок (Каждому региону соответствует папка с кодом региона в названии)
	- Используется настраиваемая пакетная загрузка в БД
	- Чтение осуществляется из ZIP архива без распаковки
	- Поддержка дельт не реализована в связи с особенностями их структуры:
		- Дельты могут содержать дублирующиеся идентификаторы.
		- Потребность в UPSERT логике для наката обновлений.
		- Есть различия в схемах первичных ключей между таблицами.
	- Для получения актуального снимка рекомендуется использовать полные выгрузки.
	- Репликатор корректно отработает их с высокой производительностью через COPY в PostgreSQL
	- Для демонстрации можно использовать дельту при условии что в ней не будет дублей.

Быстрый запуск
Предварительные требования

    Docker Desktop

    Docker Compose

    Выгрузка в формате "ГАР" с сайта ФНС
Запуск демо

Клонируйте репозиторий

	git clone <repository-url>
	cd gar-parser

Настройте переменные окружения

# Скопируйте и отредактируйте .env файл

	cp .env.example .env

Отредактируйте .env:

	DB_PASSWORD=your_password
	ConnectionStrings_DefaultConnection="Host=localhost;Port=5433;Database=GARBase;Username=postgres;Password=your_password"
	SourceSettings_ZipSourcePath="/data/your_data_file.zip"
	SourceSettings_Regions= [50,77] - список регионов, не задано (оставить пустым) — все регионы
	ReplicationSettings_Strategy=FullRepl
	ReplicationSettings_BatchSize=1000000

/data из корня проекта в compose смонтирована как том поэтому архив можно положить в нее
Regions допускает null, в этом случае парсинг отработает по всем xml внутри архива
Strategy - пока две FullRepl - все сущности ГАР и WithoutCHRepl - все без таблицы ChangeHistory
BatchSize - размер пачки собираемой перед записью в базу данных одной транзакцией


Быстрый старт с Docker:

В один образ включены:
- **GARReplication** - консольная утилита для репликации
	- Парсит данные из выгрузок ФНС в ZIP архиве в формате "ГАР"
	- Имеет фильтрацию по регионам, таблица с кодами регионов в корне проекта
- **PostgreSQL** — целевая база данных
	- скрипт инициализации базы присутствует в образе
	- и создаст базу при первом запуске.
- **pgAdmin** — веб-интерфейс для просмотра результатов

Перед запуском репликации:
	- Необходимо скачать выгрузку с сайта ФНС и положить в /data в корне проекта
	- ВАЖНО! Дельты могут приводить к ошибкам из за дублей первичных ключей,
		поэтому рекомендуется использовать только полную выгрузку.
После запуска контейнера можно:
1. Запустить репликацию командой: "docker-compose run --rm GARreplication-console start"
2. Открыть pgAdmin и увидеть загруженные данные
3. Проверить корректность парсинга без дополнительных настроек

Состав docker образа:

    GARreplication-console: Утилита репликации

    db: PostgreSQL база данных

    pgadmin: Веб-интерфейс для управления БД

Управление контейнерами


	# Запуск репликации
	docker-compose run --rm GARreplication-console start

	# Остановка
	docker-compose down

	# Просмотр логов
	docker-compose logs GARreplication-console
	docker-compose logs db

	# Полная пересборка
	docker-compose down -v
	docker-compose up --build


Важные заметки:

    Использование дельт для репликации может приводить к ошибкам из за дублей первичных ключей,
	поэтому рекомендуется использовать только полную выгрузку

    Все пути к файлам данных указываются в .env
    По умолчанию для исходных данных предназначена папка /data в корне проекта

    Данные БД сохраняются в Docker volume postgres_data
    перед повторным запуском репликации требуется очистка postgres_data

    pgAdmin может запускаться некоторое время поэтому будет доступен не сразу

Решение проблем:

Приложение не подключается к БД:

    Убедитесь, что БД полностью запустилась: docker-compose logs db

    Проверьте пароль в .env файле

Нужно изменить конфигурацию:

    Отредактируйте .env файл
    или используйте аргументы коммандной строки.

    Перезапустите: docker-compose down && docker-compose up -d
